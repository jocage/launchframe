You are working inside a **Launchframe-generated** Next.js 16 SaaS project.

## Stack (exact versions — do not guess)

- **Next.js 16** — App Router, React 19, server components by default
- **TypeScript 5.9** — strict mode, `noUncheckedIndexedAccess`, `exactOptionalPropertyTypes`
- **Drizzle ORM 0.44** — PostgreSQL (check `drizzle.config.ts` for the driver)
- **Better Auth 1.3** — email/password, session cookies, edge middleware
- **Billing** — provider abstraction in `src/lib/billing/` (Stripe, Polar, or both)
- **Email** — provider abstraction in `src/lib/email/` (Resend or Nodemailer)
- **Vitest 3.2** (unit, jsdom) + **Playwright 1.55** (e2e)
- **ESLint 9** flat config + **Prettier** (double quotes, no trailing commas, 88 chars)
- **Tailwind CSS 4**
- **CI** — GitHub Actions (lint, typecheck, test, build)

## Before you write code

## Workflows

### Add a new feature

1. Create the page: `src/app/my-feature/page.tsx` (server component, thin)
2. Create the logic: `src/lib/my-feature.ts`
3. Create the test: `src/lib/my-feature.test.ts`
4. If it needs protection, add to `proxy.ts` (see "Add a protected route")
5. Run `pnpm typecheck && pnpm lint:fix`

### Add a DB table

1. Create schema: `src/db/schema/my-table.ts`

```typescript
import { pgTable, text, timestamp, uuid } from "drizzle-orm/pg-core";

export const myTable = pgTable("my_table", {
  id: uuid("id").primaryKey().defaultRandom(),
  name: text("name").notNull(),
  createdAt: timestamp("created_at", { withTimezone: true }).defaultNow().notNull(),
});
```

2. Re-export from `src/db/schema/index.ts`:

```typescript
export * from "./my-table";
```

3. Run `pnpm db:generate` then `pnpm db:migrate`

### Add a protected route

Edit `proxy.ts`:
- Add the path to the route condition (e.g. `pathname.startsWith("/my-feature")`)
- Add the pattern to `config.matcher`

### Add an env var

1. Add Zod field in `src/lib/env.ts` schema
2. Add `process.env.MY_VAR` in the `parseEnv()` block
3. Add default value in `.env.example`

Always three places — never skip `.env.example`.

### Add a billing webhook handler

1. Create route: `src/app/api/webhooks/my-provider/route.ts`
2. Use the `BillingProvider` interface from `src/lib/billing/types.ts`:

```typescript
import { type WebhookInput } from "@/lib/billing/types";
```

3. Parse raw body, verify signature, delegate to provider's `handleWebhook()`
4. Return `NextResponse.json({ received: true })`

### Add an email template

1. Add a function to `src/lib/email/templates.ts`:

```typescript
export function createMyEmailTemplate(input: {
  appName: string;
  recipientName: string;
}): EmailTemplateResult {
  return {
    subject: `${input.appName}: subject here`,
    text: `Hi ${input.recipientName}, ...`,
    html: `<p>Hi ${input.recipientName},</p><p>...</p>`,
  };
}
```

2. Use it: `await sendEmail({ to, template: createMyEmailTemplate({ ... }) })`

### Add an API route

1. Create route handler: `src/app/api/my-endpoint/route.ts`

```typescript
import { NextResponse } from "next/server";
import { getSession } from "@/lib/auth-session";

export async function GET() {
  const session = await getSession();
  if (!session) return NextResponse.json({ error: "unauthorized" }, { status: 401 });
  // logic here
  return NextResponse.json({ data: "..." });
}
```

2. For POST/PUT/DELETE, parse body with Zod before using it.

## Rules

### MUST DO

- Use `env` from `@/lib/env` for all environment variables
- Server components by default; `"use client"` only for hooks or browser APIs
- Keep route files (`page.tsx`) thin — delegate to `src/lib/`
- Use `getSession()` from `@/lib/auth-session` for server-side session
- Use `authClient` from `@/lib/auth-client` for client-side auth
- Use `type` imports: `import { type Foo }` (ESLint enforces this)
- Run verification after changes (see below)

### MUST NOT DO

- Never access `process.env` outside `src/lib/env.ts` and `drizzle.config.ts`
- Never import from `@/db` or `server-only` modules in client components
- Never use the `any` type
- Never commit `.env` files or secrets
- Never skip the pre-commit hook

## Common errors

**"env.X is not defined"** — you added an env var to code but not to `src/lib/env.ts`. Add it to the Zod schema + `parseEnv()` block + `.env.example`.

**Auth redirect loop** — check `proxy.ts` matcher. Sign-in/sign-up pages must NOT be in the protected routes list.

**Drizzle migration conflict** — run `pnpm db:generate` to create a new migration, never edit existing migration files.

**Hydration mismatch** — a server component is importing client-only code. Check for `useState`, `useEffect`, or browser API usage without `"use client"` directive.

**"GITHUB_CLIENT_ID is required"** — Better Auth validates social provider config at build time. Set real values or remove the social provider from `src/lib/auth.ts`.

**"Module not found: @/db"** — you imported a server module in a `"use client"` file. Move the data fetching to a server component parent.
